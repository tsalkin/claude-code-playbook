# claude-code-playbook

**Операционные конвенции для агента Claude Code — правила, которые делают долго-живущего кодинг-агента надёжным, а не просто быстрым.**

Курируемый, готовый к копированию набор конвенций для глобального `~/.claude/CLAUDE.md` (грузится в каждую сессию любого проекта). Лицензия MIT. Это проза, не код — берите что подходит, остальное игнорируйте.

> Способный агент без операционной дисциплины сползает в «правдоподобно-но-неверно»: хватается за первую идею, выдумывает конкретику когда мало контекста, молча правит источник правды. Эти конвенции — предохранители, по большей части выученные на граблях.

> Брат-проект к **[claude-memory-hygiene](https://github.com/tsalkin/claude-memory-hygiene)** — тот держит индекс памяти агента в бюджете; этот держит дисциплинированным его *поведение*.

---

## Как пользоваться

Claude Code грузит `~/.claude/CLAUDE.md` в **каждую** сессию **каждого** проекта. Вставьте туда нужные конвенции; собственный `CLAUDE.md` проекта переопределяет/дополняет их. Это принципы, не конфиг — адаптируйте формулировки под себя.

---

## 1. Исследование до реализации — prior-art + внешний multi-LLM

Для **нетривиальных** задач/фаз research-этап включает:

- **Prior-art scan.** Искать похожие решения в web **и на GitHub** → раздел *Prior Art*: ссылки, что взять, чего избегать, заметка по лицензии. Дисциплина: **учись, не копируй**; проверь, что это реально ложится на твой стек.
- **Внешний multi-LLM research-промпт** *(по решению агента — не на каждую задачу)*. Когда решение весомо / ново / спорно / это арх-развилка — агент составляет **самодостаточный research-промпт** (фон + конкретные вопросы + желаемый формат ответа, чтобы внешняя модель без доступа к репо могла ответить), **сохраняет на диск** (`docs/research/<дата>_<тема>_research_prompt.md`), вы прогоняете его через несколько LLM вручную, и агент **синтезирует** ответы в `_SYNTHESIS` + `_DECISIONS`, которые идут в спецификацию.

Пропускать для тривиального/механического. Суть: не изобретать заново и не сходиться на первом ответе одной модели.

## 2. Уровень reasoning-усилия

`auto` — база. Поднимать **осознанно** там, где цена ошибки высока и глубина окупается: отладка хитрых багов (особенно класс *«несколько багов, один корень»*), проектирование новой подсистемы, изменения в ядре с каскадными эффектами. Держать низким на механике с известным путём (рестарты, bump версии, рутинные деплои).

## 3. Выбор и рекомендации

Любой вопрос с выбором: пояснить каждый вариант простым языком, показать последствия **1-го и 2-го порядка**, дать рекомендацию с обоснованием. Всегда включать вариант, работающий на **долгосрочное здоровье** (а не латание на скорую руку), и явно его помечать.

## 4. Инженерная дисциплина

- **Проверять свою правку end-to-end** — прогнать реальный путь и убедиться в результате; *«код написан» ≠ «работает»*.
- **Перед изменением поля найти его источник правды** — имя поля врёт; рассинхрон здесь — корень целых классов багов.
- **Никакой тихой автоматизации источника правды** — изменения ключевых данных (настройки, привязки, статусы) только по явному действию/подтверждению. Авто = невидимый сбой.
- **Секрет в чате = скомпрометирован** — немедленно отозвать и перевыпустить; новый только через терминал.
- **Платные действия — только с явным подтверждением** — *«есть ключ/доступ» ≠ разрешение*.
- **Фиксировать отложенные идеи** в backlog/notes — ничего не теряется.
- **Честно сообщать результат** — тесты упали — сказать с выводом; шаг пропущен — сказать.

## 5. Гигиена памяти

Хэндофф сессии живёт в **git-файлах проекта** (`RESUME`/`STATE`), не в каталоге памяти агента; в памяти — только durable-факты. Держать always-loaded индекс лёгким → **[claude-memory-hygiene](https://github.com/tsalkin/claude-memory-hygiene)**.

## 6. Документация и нарратив

Вести низкопороговый **журнал решений** (DEVLOG); крупные решения выносить в **article-ready deep-doc'и**; держать **нарративный слой без жаргона** для широкой аудитории. Документировать решения по горячим следам — это сырьё для статей.

---

## Зачем playbook (а не просто привычки)

Привычки не переживают сброс контекста и новые проекты. Глобальный `CLAUDE.md` — переживает. Записать конвенции один раз, глобально — вот что заставляет агента вести себя одинаково между сессиями, а не переучиваться на тех же уроках.
